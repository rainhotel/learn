# Agent Long-Term Memory Human Brief

## What This Research Is About

- 这项研究在看：AI 系统如何像“有长期记忆的智能体”一样跨会话保存、检索、更新、遗忘和强化信息。
- 现在值得看：2025 年开始，很多项目已经不再把 memory 当成“再套一层向量库”，而是开始做分层记忆、时间感知、后台整理、记忆演化和 MCP 集成。

## Current Best Understanding

- 目前最重要的发现：工程上最主流的方向不是单一 memory store，而是“核心记忆 + 事件记忆 + 语义记忆 + 后台整理”的分层方案。
- 目前最值得相信的结论：时间变化和冲突更新是长期记忆系统的硬问题，所以 2025 年最有代表性的系统开始转向 temporal graph、memory evolution、reflective retrieval。
- 目前最大的疑问：遗忘曲线到底该用于“删除记忆”，还是用于“降低权重、安排复习、决定何时压缩与再提炼”。

## Decision Value

- 这项研究能帮助我做什么判断：
- 如果要做一个可用的 agent memory，应该先做哪些层，哪些以后再加。
- 哪些开源项目适合直接接入，哪些更适合借鉴设计。
- 是否值得把“遗忘/复习调度”纳入自己的 memory engine。
- 还不能帮助我判断什么：
- 某个项目在我自己的 workload 上一定最好。
- 论文指标能否直接迁移到真实产品环境。
