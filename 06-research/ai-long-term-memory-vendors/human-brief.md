# AI 厂商长期记忆实现调研 Human Brief

## What This Research Is About

- 这项研究在看：主流 AI 厂商到底如何把“跨会话记忆”做成产品能力。
- 为什么现在值得看：长期记忆已经从聊天产品的个性化功能，扩展成 coding agent 的协作基础设施。

## Current Best Understanding

- 目前最重要的发现：
  - 市场上已经分化出两条主线：一条是“用户画像/偏好记忆”，一条是“项目/工作流记忆”。
  - `ChatGPT`、`Gemini`、`Microsoft Copilot` 更偏前者。
  - `Claude Code`、`Codex` 更偏后者，但两者做法明显不同。
- 目前最值得相信的结论：
  - `Claude Code` 的长期记忆更完整，已经具备“显式规则文件 + 自动生成的项目记忆目录”双层结构。
  - `Codex` 当前公开能力更偏“显式指令链 + 长时任务中的上下文压缩”，更强调长任务续航，而不是自动积累可审计记忆。
- 目前最大的疑问：
  - `Codex` 是否存在尚未公开的自动项目记忆层；当前官方资料没有给出与 `Claude Code auto memory` 对等的公开说明。

## Decision Value

- 这项研究能帮助我做什么判断：
  - 选型时，区分“记住我”与“记住项目”的不同产品路线。
  - 设计 agent memory 时，明确要做显式记忆、历史推断、项目规则还是上下文压缩。
  - 判断 `Codex` 和 `Claude Code` 分别适合哪类长期协作。
- 还不能帮助我判断什么：
  - 各家服务端未公开的召回排序、embedding 索引、权重更新策略。
  - 各家真实线上记忆命中率与遗忘率的精确量化。
