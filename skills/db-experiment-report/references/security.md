# 数据库安全策略

## 操作权限

本技能允许执行**所有 SQL 操作**，包括 SELECT、INSERT、UPDATE、DELETE 以及 DDL（CREATE、ALTER、DROP 等），不再限制为只读。

## 连接前的强制步骤：全库备份确认

在连接数据库之前，必须执行以下步骤。**不得跳过。**

### 1. 明确提示用户备份全库

输出以下警告信息（逐字）：

```
⚠️ 数据库安全警告

本技能现在允许执行所有 SQL 操作，包括但不限于：
  - 数据修改：INSERT / UPDATE / DELETE / MERGE / TRUNCATE
  - 结构修改：CREATE / ALTER / DROP
  - 存储过程执行：EXEC / EXECUTE

在继续之前，请确认你已完成以下操作：
  1. 对目标数据库进行了完整备份
  2. 确认备份文件可恢复
  3. 了解本次操作可能对数据产生的影响

是否已备份全库并确认继续？（回复"是"或"确认"以继续）
```

### 2. 等待用户确认

- 用户必须明确回复"是"、"确认"、"继续"或"yes"、"y"才可继续
- 如果用户回复"否"、"不"、"取消"或"no"、"n"，立即停止，不连接数据库
- 不可跳过此确认步骤，即使之前已确认过

### 3. 确认后开始连接

用户确认后，方可连接数据库并执行实验报告的各个阶段。

## 连接方式

- 使用 Windows 身份认证：`Trusted_Connection=yes`
- 主机名来自 `.env` 的 `SQL_SERVER_HOST`
- 连接字符串不使用 SQL Server 账号密码
