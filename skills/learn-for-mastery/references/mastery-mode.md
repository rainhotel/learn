# Mastery Mode Reference

## Purpose

Use this reference when planning or updating a topic as a long-horizon mastery path instead of an ad hoc Q&A track.

## Planning Stack

### Long Arc

- Target: become able to explain, build, critique, and extend work in the field
- Horizon: months to years
- Outputs: canonical outline, portfolio projects, teaching artifacts, review notes

### Medium Arc

- Target: achieve the next visible level-up in 8 to 12 weeks
- Outputs: weekly milestones, practice sequence, one meaningful project

### Session Arc

- Target: move one clear sub-skill forward in 30 to 90 minutes
- Outputs: notes, exercise, corrected misconception, next-step pointer

## Mastery Phases

### 0. Position The Field

- Define scope, boundaries, neighboring fields, and prerequisites.
- State what mastery means for this topic.
- Avoid starting with tools before defining the field.

### 1. Map The Terrain

- Build the vocabulary, branches, and major questions of the field.
- Produce a bird's-eye summary that a beginner can revisit.

### 2. Learn The Foundations

- Master definitions, primitives, invariants, and core mental models.
- Track common confusions and fragile intuitions early.

### 3. Understand The Mechanisms

- Explain how systems in the field actually work.
- Prefer causal chains, inputs, outputs, constraints, and failure modes.

### 4. Study Patterns And Trade-Offs

- Compare standard approaches, not just isolated facts.
- Record why one pattern is chosen over another.

### 5. Practice Deliberately

- Design drills that isolate weak sub-skills.
- Repeat with variation and capture error patterns.

### 6. Build Real Artifacts

- Create a smallest-real project, then a constrained intermediate project.
- Use projects to expose missing depth.

### 7. Teach And Defend

- Summarize the topic in your own words.
- Teach, write, or present it.
- Defend design choices against alternatives.

### 8. Develop Taste

- Compare classic and modern approaches.
- Track frontier trends without neglecting fundamentals.
- Build judgment, not novelty-chasing.

## Phase Template

For each phase, specify:

- Goal
- Key concepts
- Practice method
- Expected artifact
- Exit criteria
- Common traps

## Exit Criteria

Do not mark a phase complete just because material was consumed. Prefer checks like:

- Can explain without reading
- Can give examples and counterexamples
- Can apply in a constrained exercise
- Can name trade-offs
- Can detect common mistakes

## Human Priority

Always keep one human-facing summary current. The best default is a guide that answers:

- Why does this topic matter?
- What stage am I in?
- What should I do this week?
- Where do I resume next session?
