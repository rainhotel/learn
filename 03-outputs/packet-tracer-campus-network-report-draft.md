## 摘要

本文以校园网络建设为背景，基于 Cisco Packet Tracer 平台完成了一个中小型校园网络的规划、设计、配置与调试。该网络按照实际应用场景将终端划分为服务器区、实验室、教学楼和图书馆等多个逻辑区域，并通过 VLAN 技术实现二层隔离，通过核心三层交换实现不同 VLAN 之间的互联互通。同时，网络中部署了 DNS 服务和 Web 服务，并通过核心路由器、出口路由器与外部网络建立连接，使校园内部主机能够访问外网主机和服务器。设计过程中对交换机、路由器、服务器等设备进行了详细配置，并通过多组连通性测试验证了网络的正确性。结果表明，该方案能够较好地满足校园网络在资源共享、分层管理、服务部署和外网访问等方面的基本需求，具有一定的可扩展性和工程实践意义。

**关键词：** Packet Tracer；校园网络；VLAN；三层交换；静态路由；DNS

## 1 需求分析

### 1.1 项目背景

随着校园信息化建设的不断推进，教学、实验、办公和资源服务对网络提出了越来越高的要求。传统的单一平面网络在管理、维护、安全隔离和扩展能力方面存在明显不足，因此需要构建一个结构清晰、功能完整、便于管理的分层网络系统。

本次课程设计以校园网络为应用场景，要求在 Cisco Packet Tracer 环境中完成一个中小型网络的设计与实现。设计目标是在模拟环境下完成网络基础设施搭建、地址规划、交换与路由配置、基础网络服务部署以及内外网连通性验证，从而达到将理论知识与工程实践结合的目的。

### 1.2 设计目标

本网络系统的主要设计目标如下：

1. 采用分层思想构建校园网络拓扑，体现接入层、核心交换层和路由出口层的结构特点。
2. 对不同业务区域进行 VLAN 划分，提高网络管理效率并增强广播域隔离能力。
3. 通过核心三层交换机实现不同 VLAN 之间的互通。
4. 在服务器区部署 DNS 服务器和 Web 服务器，支持基础网络服务访问。
5. 实现校园网与外部网络之间的互联，使内部用户可以访问外网主机。
6. 在拓扑中加入冗余链路，提高网络结构的完整性和可靠性。

### 1.3 功能需求分析

结合本次设计的拓扑结构，校园网络应满足以下功能需求：

1. 服务器区、实验室、教学楼、图书馆应分别属于不同的逻辑网段。
2. 各接入交换机下的主机能够正确接入所属 VLAN。
3. 不同 VLAN 的主机之间能够通过核心交换机实现三层通信。
4. DNS 服务器能够提供域名解析服务。
5. 校园 Web 服务器能够为内网用户提供网页访问服务。
6. 校园内部主机能够访问右侧外部网络中的主机和服务器。
7. 外部网络中的主机能够通过出口路由访问校园网络中的相关资源。

### 1.4 技术需求分析

为满足上述功能，本系统主要采用以下网络技术：

1. **VLAN 技术**  
   利用 VLAN 将不同业务区域进行逻辑划分，减少广播域范围，增强网络管理能力。

2. **Trunk 中继技术**  
   在核心交换机与接入交换机之间建立 Trunk 链路，使多 VLAN 数据能够在交换机之间传递。

3. **三层交换技术**  
   在核心交换机上配置 VLAN 接口（SVI）并开启 `ip routing`，实现不同 VLAN 之间的通信。

4. **静态路由技术**  
   在核心路由器、出口路由器之间配置静态路由，明确不同网段的到达路径，保证内外网互访。

5. **DNS 服务与 Web 服务**  
   在服务器区配置 DNS 服务器与 Web 服务器，用于模拟校园网络中常见的基础应用服务。

### 1.5 服务需求分析

本次设计中主要部署以下服务：

1. **DNS 服务**  
   为内网主机提供域名解析能力，使用户可以通过域名访问服务器，而不必记忆具体 IP 地址。

2. **WWW 服务**  
   在服务器上部署 Web 服务，用于模拟校园信息发布、教学资源共享等典型应用。

### 1.6 网络可靠性需求

为了提高网络的完整性与工程合理性，本设计在核心层设置了多条互联链路，使网络具备一定的冗余结构。即使某一条链路出现故障，网络仍可通过其他路径维持部分通信能力。这种设计思想与实际网络工程中的高可用思路相一致。

## 2 概要设计

### 2.1 总体设计思路

本次网络采用典型的层次化设计思想，整体上可以划分为接入层、核心交换层、核心路由层和外网区域四个部分。

1. **接入层**  
   由多台 2960 交换机构成，分别连接不同区域的主机或服务器，负责终端接入。

2. **核心交换层**  
   由两台 3560 三层交换机构成，负责 VLAN 汇聚、Trunk 中继和 VLAN 间路由转发。

3. **核心路由层**  
   由多台 2811 路由器组成，负责核心网络转发、出口路由和冗余链路互联。

4. **外网区域**  
   由出口路由器、交换机、外网 PC 和外网服务器组成，用于模拟校园网络访问互联网的场景。

### 2.2 网络拓扑结构

本网络主要由以下设备构成：

1. 两台核心交换机：`CORE Switch 1`、`CORE Switch 2`
2. 两台核心路由器：`CORE-Router 1`、`CORE-Router 2`
3. 一台出口汇聚路由器：`CK-Router`
4. 一台外网路由器：`P-Router`
5. 四台接入交换机：分别对应 VLAN10、VLAN20、VLAN30、VLAN40
6. 两台校园服务器：`DNS-Server`、`www.niust.edu`
7. 一台外网用户主机和一台外网服务器：`www.baidu.com`

### 2.6 功能实现思路

本系统功能的实现过程如下：

1. 在核心交换机上创建 VLAN10、VLAN20、VLAN30、VLAN40。
2. 在核心交换机上配置各 VLAN 的 SVI 接口地址作为默认网关，并开启三层交换功能。
3. 在核心交换机与接入交换机之间配置 Trunk 中继口，允许多个 VLAN 通过。
4. 在接入交换机上为终端接口配置 Access 模式并划入对应 VLAN。
5. 在核心路由器和出口路由器之间配置 IP 地址与静态路由，实现内外网互通。
6. 在服务器上配置 DNS、WWW 服务并进行访问测试。

## 3 详细设计

### 3.1 核心交换机设计

#### 3.1.1 CORE-SW1 的作用

`CORE-SW1` 是本次设计中的关键设备之一，其主要功能包括：

1. 创建并维护 VLAN10、VLAN20、VLAN30、VLAN40。
2. 为各 VLAN 提供三层网关地址。
3. 通过 `ip routing` 实现不同 VLAN 之间的互通。
4. 与接入交换机通过 Trunk 链路进行 VLAN 中继。
5. 与核心路由器建立三层接口，完成校园网出口转发。

#### 3.1.2 CORE-SW1 关键配置

以下为核心交换机的关键配置思路：

```cisco
vlan 10
vlan 20
vlan 30
vlan 40

interface vlan 10
 ip address 192.168.10.254 255.255.255.0
 no shutdown

interface vlan 20
 ip address 192.168.20.254 255.255.255.0
 no shutdown

interface vlan 30
 ip address 192.168.30.254 255.255.255.0
 no shutdown

interface vlan 40
 ip address 192.168.40.254 255.255.255.0
 no shutdown

ip routing
```

代码 1 说明：  
以上配置完成了 VLAN 的创建、各 VLAN 网关地址的设置，以及三层交换功能的开启。这样各 VLAN 主机在将默认网关配置为 `.254` 后，即可通过三层交换机实现跨 VLAN 通信。

```cisco
interface range fa0/3 - 8
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
 no shutdown
```

代码 2 说明：  
该配置将核心交换机连接接入层和其他核心设备的端口设置为 Trunk 模式，允许 VLAN10、20、30、40 的报文通过。这样多个 VLAN 的数据就可以在交换机之间传递。

#### 3.1.3 CORE-SW2 的作用

`CORE-SW2` 主要用于增强网络结构完整性和冗余性，并承担部分上联与互联功能。在本设计中，它与 `CORE-SW1` 共同组成核心交换层，并与核心路由器建立连接。

### 3.2 接入交换机设计

接入交换机分别对应不同业务区域，其主要作用是：

1. 为终端主机提供二层接入。
2. 将终端所在端口配置为 Access 模式。
3. 将与核心交换机相连的端口配置为 Trunk 模式。

#### 3.2.1 Switch0（VLAN10 服务器区）

服务器区交换机主要连接 DNS 服务器和 Web 服务器，因此其终端口属于 VLAN10，上联核心交换机的端口为 Trunk 模式。

```cisco
vlan 10

interface range fa0/1 - 2
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
 no shutdown

interface range fa0/3 - 4
 switchport mode access
 switchport access vlan 10
 no shutdown
```

代码 3 说明：  
该配置保证了服务器区交换机能够一方面承载上联 Trunk 流量，另一方面让接入的服务器正确加入 VLAN10。

#### 3.2.2 Switch1（VLAN20 实验室）

实验室交换机的用户端口配置为 VLAN20，其上联端口为 Trunk。

```cisco
vlan 20

interface range fa0/1 - 2
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
 no shutdown

interface fa0/3
 switchport mode access
 switchport access vlan 20
 no shutdown
```

#### 3.2.3 Switch2（VLAN30 教学楼）

教学楼交换机的终端口属于 VLAN30，用于承载教学楼区域主机接入。

```cisco

interface range fa0/1 - 2
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
 no shutdown

interface fa0/3
 switchport mode access
 switchport access vlan 30

```

#### 3.2.4 Switch3（VLAN40 图书馆）

图书馆交换机的终端口属于 VLAN40，用于连接图书馆用户主机。

```cisco

 switchport trunk allowed vlan 10,20,30,40
 no shutdown

interface fa0/3

 switchport access vlan 40

```

### 3.3 核心路由器与出口路由器设计

#### 3.3.1 CORE-Router 1

`CORE-Router 1` 作为主路径核心路由设备，承担以下作用：

1. 与 `CORE-SW1` 建立三层连接。
2. 与 `CK-Router` 建立出口链路。
3. 通过静态路由将来自校园网的流量转发到出口方向。
4. 将回程流量转发回校园三层交换机。

关键配置示例如下：

```cisco
interface fa1/0
 ip address 192.168.254.1 255.255.255.0
 no shutdown

interface fa0/0
 ip address 10.0.12.1 255.255.255.252
 no shutdown

ip route 0.0.0.0 0.0.0.0 10.0.12.2
ip route 192.168.0.0 255.255.0.0 192.168.254.2
```

代码 4 说明：  
该配置使 `CORE-Router 1` 既能把校园网流量转发至出口路由方向，也能将返回的校园网流量正确交还给核心交换机。尤其是回程静态路由的补充，是解决左侧 PC 无法访问右侧网络的关键步骤之一。

#### 3.3.2 CK-Router

`CK-Router` 位于校园核心路由层与外部网络之间，是整个拓扑中的出口汇聚节点。其主要任务是：

1. 与内部核心路由器建立链路；
2. 与外部路由器 `P-Router` 通过串口建立连接；
3. 为校园网与外部网络之间提供转发通道。

关键配置示例如下：

```cisco
interface fa0/0
 ip address 10.0.12.2 255.255.255.252
 no shutdown

interface serial0/0/0
 ip address 200.10.1.1 255.255.255.240
 clock rate 64000
 no shutdown

ip route 192.168.0.0 255.255.0.0 10.0.12.1
ip route 100.10.1.0 255.255.255.240 200.10.1.2
```

#### 3.3.3 P-Router

`P-Router` 用于模拟外部网络出口设备，其作用是为右侧 `100.10.1.0/28` 网段提供网关，并与 `CK-Router` 互联。

```cisco
interface serial0/0/0
 ip address 200.10.1.2 255.255.255.240
 no shutdown

interface fa0/1
 ip address 100.10.1.1 255.255.255.240
 no shutdown

ip route 192.168.0.0 255.255.0.0 200.10.1.1
```

代码 5 说明：  
以上配置使 `P-Router` 能够一方面作为外网用户区的默认网关，另一方面通过静态路由将访问校园网的流量转发给 `CK-Router`。

该表展示了外网用户区的地址规划。其中 `100.10.1.1` 由 `P-Router` 提供，作为外网用户区的默认网关。

### 3.5 冗余链路设计说明

本次设计中在核心交换机和核心路由器之间设置了多条互联链路。这些链路的主要目的并不是改变主业务流程，而是模拟实际工程中常见的冗余结构。

在调试过程中，冗余链路涉及如下处理：

1. 为新增链路规划新的独立网段，避免地址重复。
2. 对接口执行 `no shutdown`，保证端口激活。
3. 对交换机三层接口执行 `no switchport`。
4. 使用 `show ip interface brief` 检查接口是否处于 `up/up` 状态。

1. 首先验证接入层主机能否与本 VLAN 网关通信；
2. 然后验证不同 VLAN 之间能否互通；

4. 最后验证校园网主机能否访问外部网络主机与服务器。

这种由近及远、由简单到复杂的调试方法能够更快定位问题，也符合实际网络工程中的排错流程。

### 4.6 典型问题与解决过程

在本次调试过程中，出现了若干典型问题，解决过程如下：

#### （1）Trunk 配置报错

问题现象：  
在 3560 核心交换机上配置 `switchport mode trunk` 时，提示端口封装模式为 `Auto`，无法直接切换为 Trunk。

原因分析：  
3560 交换机部分端口默认采用自动封装协商模式，需要先显式指定封装类型。

解决方法：  
先执行：

```cisco
switchport trunk encapsulation dot1q
```

再执行：

```cisco
switchport mode trunk
```

#### （2）接口状态为 administratively down

问题现象：  
部分路由器或交换机接口已配置地址，但仍无法通信。

原因分析：  
接口处于关闭状态，未执行 `no shutdown`。

解决方法：  
进入对应接口模式，执行：

```cisco
no shutdown
```

#### （3）路由器没有串口接口

问题现象：  
在配置 `serial0/0/0` 时，系统提示接口不存在。

原因分析：  
路由器未安装串口模块。

解决方法：  
在 Physical 模式下关闭设备电源，插入 `WIC-2T` 串口模块，重新开机后再进行配置。

#### （4）右侧网关接口 down/down

问题现象：  
`P-Router` 的 `Fa0/1` 接口一直处于 `down/down` 状态。

原因分析：  
交换机与路由器之间连线端口不匹配，或者未使用正确的双绞线。

解决方法：  
重新选择 `Copper Straight-Through` 连接路由器与交换机，并确认交换机端实际连接端口为指定的 `Fa0/1`。

#### （5）左侧 PC 无法访问右侧外网主机

问题现象：  
右侧本地通信正常，但左侧校园网主机最初无法 ping 通右侧外网主机。

原因分析：  
`CORE-Router 1` 缺少返回校园网各 VLAN 的静态路由，导致回程路径不完整。

解决方法：  
在 `CORE-Router 1` 上补充：

```cisco
ip route 192.168.0.0 255.255.0.0 192.168.254.2
```

这一步是实现最终内外网互通的关键配置。

3. DNS 服务器和 Web 服务器完成地址配置与服务部署；
4. 核心交换机、核心路由器、出口路由器之间实现静态路由转发；
5. 校园内部主机可以成功访问右侧外部网络主机与服务器；
6. 网络整体功能满足课程设计要求。

## 5 总结与心得

通过本次课程设计，我系统地完成了一个中小型校园网络从需求分析、拓扑规划、地址设计、交换配置、路由配置、服务部署到调试验证的全过程。在设计过程中，我不仅加深了对 VLAN、Trunk、三层交换、静态路由、DNS 和 Web 服务等知识点的理解，也进一步认识到网络工程实践中“规划先行、逐层验证、逐段排错”的重要性。

本次实验中，最具有代表性的收获主要有以下几点：

1. **网络设计必须先进行清晰的地址规划。**  
   若地址规划不统一，后续配置极易冲突，尤其在冗余链路较多时更容易产生混乱。

2. **交换与路由问题的排查方法不同。**  
   交换层更关注 VLAN、Trunk 和接口模式，路由层更关注 IP 地址、接口状态和路由表。

3. **静态路由不仅要考虑去程，也要考虑回程。**  
   本次调试中，虽然出口设备已能访问校园网，但校园主机访问外网失败，根本原因就在于回程路由不完整。

4. **Packet Tracer 能较真实地模拟工程调试过程。**  
   例如接口 `up/down`、`administratively down`、`down/down` 等状态，都能够帮助定位问题来源。

总的来说，本次课程设计使我将书本中的理论知识落实到具体网络工程场景中，提高了独立分析问题和解决问题的能力，也为后续学习更复杂的网络协议与工程实践打下了基础。
