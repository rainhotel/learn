# 计组实验6：MIPS 寄存器文件设计（可直接用于报告）

## 1. 实验目的

理解 MIPS 寄存器文件（Regfile）的结构与工作方式，掌握在 Logisim 中使用寄存器、分线器、多路选择器等器件完成“2读1写”寄存器堆设计的方法。

## 2. 实验任务

在 Logisim 中完成 `◇MIPS Regfile` 子电路，功能要求：

1. 包含 4 个 32 位寄存器（编号 0~3）。
2. 两个读端口：输入 `R1#`、`R2#`，输出 `Reg1Out`、`Reg2Out`。
3. 一个写端口：`Write Reg#`、`Din`、`Write Enable`、`Clock`。
4. 0 号寄存器恒为 0（不可被写入非零值）。
5. 地址输入是 5 位，但本实验只使用低 2 位（高 3 位忽略）。

## 3. 实验步骤、结果与分析

### 3.1 打开工程并定位子电路

1. 打开文件 [storage.circ](D:/moniC/2026-6寄存器实验 - 副本/storage.circ)。
2. 在左侧电路列表中进入 `◇MIPS Regfile`。
3. 确认引脚名称与位宽：
   - 输入：`R1#(5)`、`R2#(5)`、`Write Reg#(5)`、`Din(32)`、`Write Enable(1)`、`Clock(1)`
   - 输出：`Reg1Out(32)`、`Reg2Out(32)`

建议截图：`◇MIPS Regfile` 总体电路图（作为“图1：Regfile 总体结构”）。

### 3.2 核心电路实现逻辑

1. 地址处理：用 3 个 Splitter 取 `R1#`/`R2#`/`Write Reg#` 低 2 位作为寄存器编号。
2. 写通路：
   - `Din` 连接写数据总线。
   - 写地址低 2 位选择目标寄存器。
   - `Write Enable` 与时钟共同控制写入，仅在有效时写入。
   - 0 号寄存器保持常零（不允许写入非零）。
3. 读通路：
   - 两个 4选1、32位 MUX，分别根据 `R1#` 与 `R2#` 选择读数据。
   - 输出到 `Reg1Out`、`Reg2Out`。

建议截图：写通路和读通路局部放大图（作为“图2：写入控制逻辑”“图3：双读端口选择逻辑”）。

### 3.3 功能测试

1. 进入 `◆MIPS Regfile测试` 子电路。
2. 启动时钟运行自动测试逻辑。
3. 观察测试完成指示灯、分数显示和错误寄存器追踪显示。
4. 若失败，回到 `◇MIPS Regfile` 检查：
   - 地址是否只取低 2 位；
   - 0 号寄存器是否被错误写入；
   - 两路 MUX 选择线是否接反；
   - 时钟、写使能极性是否正确。

### 3.4 测试现象说明（报告可写）

自动测试按“写入遍历 + 双端口读取比对”方式验证寄存器文件。通过时，测试完成信号拉高，错误追踪无异常，得分显示正常。  
调试中最常见问题是地址位截取错误和 0 号寄存器未锁零。

## 4. 分析与讨论（可直接使用）

### 4.1 存储系统什么时候需要字扩展、位扩展？

1. 位扩展：当单个存储器芯片“字长不够”时，需要把多个芯片并联扩展数据位宽。  
   例：用 8 位芯片组成 32 位数据通路，需要 4 片并联。
2. 字扩展：当“可寻址字数不够”时，需要扩展地址深度。  
   例：每片只有 1K 字，但系统要 4K 字，就要做字扩展。
3. 实际系统经常同时使用字扩展和位扩展，以满足容量与带宽要求。

### 4.2 寄存器常用引脚有哪些？引脚个数与什么有关？

常用引脚包括：

1. 数据输入 `Din`
2. 数据输出 `Dout`（或多读端口 `RD1/RD2`）
3. 写使能 `WE`
4. 时钟 `CLK`
5. 地址引脚（读地址、写地址）
6. 可选控制（清零、复位、片选等）

引脚个数主要取决于：

1. 寄存器位宽（位宽越大，数据引脚越宽）
2. 寄存器数量（地址位数 = `log2(寄存器个数)`）
3. 端口数（几读几写决定地址/数据端口个数）
4. 是否带额外控制功能（复位、使能等）

## 5. 附录（可选）

### 5.1 本实验调试要点

1. 先确保写通路正确，再检查读通路。
2. 先单步时钟验证写入，再开自动测试。
3. 一旦出现读值异常，优先检查 MUX 的选择线位序。
