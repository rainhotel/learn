# Dynamic Programming Q&A

## Open Questions

- [ ] 什么时候应该优先写记忆化搜索，再改成递推 DP？
- [ ] 一维滚动数组的成立条件需要单独总结。

## Error Patterns

- 最近反复出现的错误：把前驱坐标写反，导致转移方向错位
- 最近最容易混淆的概念：当前状态由“前驱”转移而来，不是由“后继”反推过去

## Answered Questions

### 为什么这题用 DP 很自然

- 结论：因为当前位置的答案只依赖更小坐标的位置，而且这些子问题会反复出现。
- 依据：到 `(i,j)` 的路径，只可能来自两个固定前驱。
- 例子：`dp[4][4]` 只用看 `dp[3][2]` 和 `dp[2][3]`。

### 为什么起点要设成 1

- 结论：`dp[1][1] = 1` 表示“什么都不做就站在起点”算 1 种合法起始状态。
- 依据：所有后续状态都要从它递推出来。
- 例子：如果把它设成 `0`，那整张表都会是 `0`。

### 这题能不能不用 DP

- 结论：可以用组合计数做，但 DP 更通用、更适合作为入门模板。
- 依据：每条路径本质上是两种步法的排列。
- 例子：后续若加入障碍点，组合思路会立刻变麻烦，而 DP 只需改边界或禁用点。
