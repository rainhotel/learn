# Dynamic Programming Notes

## Key Concepts

### 状态定义

- 定义：`dp[i][j]` 必须先说清楚“它表示什么”，比如“从起点走到 `(i,j)` 的方案数”。
- 为什么重要：状态定义一旦清楚，转移和答案位置就不容易写乱。
- 常见误区：只记住公式，没说清下标含义，结果一换题就不会写。

### 转移与遍历顺序

- 定义：转移就是把当前状态拆成更小状态的组合。
- 为什么重要：DP 的本质不是表格，而是依赖关系。
- 常见误区：前驱和后继写反，或者遍历顺序早于依赖准备好。

### 边界条件

- 定义：最小子问题的已知答案，例如本题的 `dp[1][1] = 1`。
- 为什么重要：没有边界，整个递推链就没有起点。
- 常见误区：把起点漏掉，或者让起点继续被公式重复累加。

### 可达性快速判断

- 定义：本题每走一步，横纵坐标之和都会增加 `3`。
- 为什么重要：如果 `i + j - 2` 不是 `3` 的倍数，那么 `(i,j)` 一定不可达。
- 常见误区：把这个观察当成主解法，忽略了 DP 才是更通用的模板。

## Mental Model

- 做计数 DP 时，优先问自己一句：当前这个位置的最后一步能从哪里来。
- 只要前驱状态都更“小”，就能考虑递推。

## Examples

- 例子 1：P1796 中，`dp[i][j] = dp[i - 1][j - 2] + dp[i - 2][j - 1]`。
- 例子 2：样例 `4 4` 中，`dp[4][4] = dp[3][2] + dp[2][3] = 1 + 1 = 2`。
- 例子 3：`2 2` 不可达，因为总增量为 `2`，不是 `3` 的倍数。

## Promoted From Problems Or Research

- 从解题中沉淀出的知识点：方案数 DP 常常是“所有合法前驱之和”。
- 从解题中沉淀出的知识点：边界和遍历顺序往往比公式本身更容易出错。
- 从研究中沉淀出的知识点：当前暂无。
