# 图像直方图处理实验大纲

## Phase 0: 任务拆解

- 明确实验 1 到 5 分别要交什么结果。
- 确认报告硬性要求：前后图像、前后直方图、算法代码、运行截图、实验总结、6 页以内。
- 准备图像：`eight.tif`、`rice.png`、`Fig0338(a)(blurry_moon)`、`pout.tif`、`Fig0809(a)`。

## Phase 1: 二值化与阈值选择

- 先看原图和直方图。
- 再利用直方图谷底人工选择阈值。
- 再使用 `graythresh` 求 Otsu 最优阈值。
- 重点输出：
  - 原图
  - 原图直方图
  - 二值化结果
  - 人工阈值与 Otsu 阈值对比结论

## Phase 2: 直方图均衡化

- 理解均衡化本质是用累计分布函数做灰度映射。
- 自己编程实现 `histeq` 的核心功能。
- 用 `pout.tif` 展示处理前后的图像与直方图变化。
- 重点输出：
  - 原图
  - 原图直方图
  - 均衡化图像
  - 均衡化后直方图

## Phase 3: 直方图规定化

- 理解规定化不是“拉平”，而是“向指定直方图靠拢”。
- 以 `Fig0809(a)` 的直方图作为目标分布。
- 用 `histeq(I, hgram)` 完成规定化。
- 重点输出：
  - 源图像和直方图
  - 参考图像或其直方图
  - 规定化结果和结果直方图

## Phase 4: 自适应直方图均衡化

- 了解全局均衡化和局部均衡化的差异。
- 查询 Matlab 中 `adapthisteq` 的功能与常用参数。
- 用同一幅测试图展示局部对比度增强效果。
- 重点输出：
  - 原图
  - 原图直方图
  - CLAHE 结果
  - CLAHE 后直方图

## Phase 5: 报告组装

- 每个实验都补“原理一句话 + 步骤一句话 + 结果一句话 + 结论一句话”。
- 把代码截图或关键代码块放入正文。
- 最后删除题目里要求删除的红色提示文字。

## Phase Exit Criteria

### 实验执行完成前

- 三幅图像都完成人工阈值与 Otsu 对比。
- `pout.tif` 的手写均衡化代码能正常运行。
- 已得到规定化和 CLAHE 的结果图。

### 报告提交前

- 所有任务都展示前后图像。
- 直方图类任务都展示前后直方图。
- 已写实验总结与体会。
- 全文控制在 6 页以内。
