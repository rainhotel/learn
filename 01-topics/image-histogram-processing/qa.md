## Error Patterns

- 最近反复出现的错误：把 `graythresh` 的结果误当成 0 到 255 的整数阈值。
- 最近最容易混淆的概念：直方图均衡化和直方图规定化。

## Answered Questions

### `graythresh` 的功能是什么？

- 结论：`graythresh` 用 Otsu 方法返回归一化全局阈值，范围通常在 `[0, 1]`。
- 依据：它通过最大化前景与背景的类间方差来选阈值。
- 例子：`level = graythresh(I); BW = imbinarize(I, level);`

### 人工阈值为什么可能优于 Otsu？

- 结论：当图像直方图不是明显双峰，或先验上知道前景/背景位置时，人工阈值可能更贴合目标。
- 依据：Otsu 只利用全局统计，不理解任务语义，也不考虑空间结构。
- 例子：模糊月亮图像可能因为灰度重叠，导致 Otsu 切分得不够理想。

### `histeq(I, hgram)` 为什么能做直方图规定化？

- 结论：因为它会根据给定目标直方图构造近似映射，使输出分布尽量接近该目标。
- 依据：本质上仍是通过累计分布函数之间的对应关系做离散映射。
- 例子：把 `pout.tif` 调整为接近 `Fig0809(a)` 的灰度分布。

### 自适应均衡化和全局均衡化怎么选？

- 结论：全局均衡化适合整体对比度不足的图像；自适应均衡化更适合局部亮度差异大的图像。
- 依据：CLAHE 在局部块上增强对比度，能补偿局部阴影和亮度不均。
- 例子：如果一张图左亮右暗，全局均衡化往往顾此失彼，CLAHE 通常更稳。
