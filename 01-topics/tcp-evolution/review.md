# TCP 发展历史和发展历程 Review

## What Went Well

- 选题主线比较清晰，容易从“问题推动演进”来组织报告。
- AI 对话式学习非常适合这个题目，因为可以逐轮提炼知识点。
- TCP 与 Web 应用的关系天然适合写“从协议到系统”的分析。

## What Was Hard

- 容易把 HTTP 层改进和 TCP 层改进写混。
- 容易只写“有哪些 RFC”，却没有说明每次修改的动机。

## What To Improve Next Time

- 如果时间允许，可以补抓包实验，让报告更有“实验味道”。
- 可以额外补一个“TCP 与 QUIC 对比”的展望段，增强结尾层次。
