# TCP 发展历史和发展历程 Formula Sheet

## 1. 吞吐近似关系

- 常见直觉：
  - RTT 越大，建立连接和恢复丢包的等待成本越高
  - 丢包率越高，传统 TCP 吞吐越容易下降
- 什么时候用：
  - 解释为什么高时延网络促使 TCP 继续演进

## 2. 带宽时延积

- 公式：
  - BDP = Bandwidth x RTT
- 含义：
  - 表示一条链路在一个往返时延内可容纳的数据量
- 什么时候用：
  - 解释为什么原始 TCP 窗口在高速长距离链路上不够大

## 3. 有效页面加载时延直觉

- 近似直觉：
  - 页面加载时延 = 建连时延 + 请求响应时延 + 资源串并行调度代价
- 什么时候用：
  - 解释 HTTP/1.0 为什么受频繁 TCP 建连拖累
  - 解释为什么 Keep-Alive、Fast Open、HTTP/2 对 Web 性能重要

## 4. TCP 关键机制速记

- 三次握手：
  - 建立连接并同步初始序号
- 滑动窗口：
  - 控制未确认数据量
- 慢开始：
  - 初期试探网络容量
- 拥塞避免：
  - 在接近容量时更平缓增长
- 快重传：
  - 不必等超时即可重传
- 快恢复：
  - 丢包后避免回到过低发送速率
- SACK：
  - 选择性确认已收到的数据块
