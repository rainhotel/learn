# TCP 发展历史和发展历程 Solved Problems

## Problem 1

- 来源：课程设计自拟问题
- 题目：TCP 在 Web 应用的发展历程中，做了哪些针对性的修改？

### Answer

- 第一类是面向网络稳定性的修改：拥塞控制、快重传、快恢复，让 TCP 在网络负载增长后仍能稳定工作。
- 第二类是面向高性能链路的修改：窗口扩大、时间戳、SACK，让 TCP 在高带宽高时延和多丢包场景下保持效率。
- 第三类是面向 Web 短流和低时延需求的修改：更大的初始拥塞窗口、Fast Open，以及 HTTP 对 TCP 持久连接和单连接多流的更充分利用。

### Steps

1. 先按时间线分层，而不是平铺所有 RFC。
2. 再把每个阶段对应到网络问题。
3. 最后从 Web 访问模式角度解释这些改进为什么重要。

### Methods

- 时间线法
- 问题驱动法
- 协议层与应用层联动分析法

### Conclusion

- TCP 的发展不是单一算法升级，而是随着互联网规模、链路条件和 Web 负载形态变化不断演进的结果。

## Problem 2

- 来源：课程设计自拟问题
- 题目：在分布式高可用高并发的 Web 应用中，TCP 起到了什么作用？

### Answer

- TCP 是可靠通信底座。
- TCP 是长连接、连接池、负载均衡和反向代理的连接基础。
- TCP 的拥塞控制帮助系统在压力下自动收敛速率，减少网络层级的雪崩风险。
- TCP 的局限也会反过来影响 Web 设计，例如 HTTP/2 在 TCP 丢包时仍会受整连接阻塞影响。

### Conclusion

- TCP 在现代分布式 Web 中既是基础设施，也是性能边界的一部分。
