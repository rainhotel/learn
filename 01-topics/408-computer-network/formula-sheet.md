# 408 计算机网络 Formula Sheet

## Network Performance

### 时延分解

- 名称：总时延
- 表达式/方法：总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延
- 含义：一个分组从发送到接收经历的总耗时
- 什么时候用：分析链路时间开销题
- 不该在什么时候用：题目只比较单一时延因素时
- 常见误用：把发送时延和传播时延混为一谈

### 发送时延

- 名称：发送时延
- 表达式/方法：发送时延 = 分组长度 / 发送速率
- 含义：把比特推上链路所需时间
- 什么时候用：计算链路发完一个帧/分组所需时间
- 常见误用：误认为和传播距离有关

### 传播时延

- 名称：传播时延
- 表达式/方法：传播时延 = 信道长度 / 电磁波传播速率
- 含义：信号在介质中传播所需时间
- 什么时候用：长距离链路题
- 常见误用：误认为和数据长度有关

## Physical Layer

### 奈奎斯特定理

- 名称：奈奎斯特定理
- 表达式/方法：理想低通信道极限码元速率 = 2W Baud
- 含义：无噪声条件下码元传输上限
- 什么时候用：无噪声理想信道题
- 不该在什么时候用：考虑噪声极限时单独使用
- 常见误用：直接当作信道容量公式

### 香农定理

- 名称：香农定理
- 表达式/方法：C = W log2(1 + S/N)
- 含义：有噪声信道的极限信息传输速率
- 什么时候用：噪声信道容量题
- 常见误用：S/N 单位和对数量级处理错误

## Data Link Layer

### CRC 核心步骤

- 名称：CRC
- 表达式/方法：数据后补 r 个 0，再模 2 除以生成多项式
- 含义：差错检测而非纠错
- 什么时候用：CRC 余数计算题
- 常见误用：普通除法替代模 2 除法

### 最小帧长

- 名称：CSMA/CD 最小帧长
- 表达式/方法：最小帧传输时延 >= 2 倍单程传播时延
- 含义：确保发送端发送未结束前可检测到冲突
- 什么时候用：以太网争用期相关题
- 常见误用：不理解它和冲突检测的因果关系

## Network Layer

### 主机数

- 名称：可用主机数
- 表达式/方法：2^h - 2
- 含义：h 位主机位时可分配主机地址数量
- 什么时候用：子网划分题
- 常见误用：漏减网络地址和广播地址

### 分片偏移量

- 名称：分片偏移量
- 表达式/方法：偏移量 = 该片数据部分起始字节号 / 8
- 含义：IP 分片中相对原数据报数据部分的偏移
- 什么时候用：IP 分片题
- 常见误用：忘记按 8 字节为单位

### 最长前缀匹配

- 名称：最长前缀匹配
- 表达式/方法：在所有匹配路由项中选前缀最长者
- 含义：CIDR 环境下路由转发原则
- 什么时候用：路由表转发题
- 常见误用：只按先后顺序匹配

## Transport Layer

### TCP 确认号

- 名称：确认号
- 表达式/方法：ACK = 期望收到的下一个字节序号
- 含义：累计确认
- 什么时候用：TCP 序号确认号题
- 常见误用：把确认号当成已收到最后一个字节序号

### 停止等待利用率

- 名称：停止等待信道利用率
- 表达式/方法：U = 发送时延 / (发送时延 + RTT + 确认处理相关时间)
- 含义：停止等待协议效率估算
- 什么时候用：链路层或运输层效率题
- 常见误用：忽略往返等待

### GBN 发送窗口上界

- 名称：GBN 发送窗口上界
- 表达式/方法：`W_T <= 2^k - 1`
- 含义：帧序号有 `k` 位时，GBN 发送窗口最大值
- 什么时候用：求帧序号位数或判断窗口是否合法
- 常见误用：把 GBN 和 SR 的上界混为一谈

### SR 窗口上界

- 名称：SR 窗口上界
- 表达式/方法：`W_T <= 2^(k-1)`，`W_R <= 2^(k-1)`
- 含义：为避免新旧帧混淆，SR 的发送/接收窗口都不能超过一半序号空间
- 什么时候用：SR 窗口题
- 常见误用：误套 GBN 的 `2^k - 1`

### 满利用率窗口判定

- 名称：滑动窗口满利用率判定
- 表达式/方法：`W * Td >= Td + 2Tp + Tack`
- 含义：窗口内帧发完前，第一帧确认应已返回
- 什么时候用：GBN/SR 利用率与最小窗口题
- 常见误用：忽略确认帧发送时延

### 平均传输速率

- 名称：窗口协议平均传输速率
- 表达式/方法：`吞吐率 = 发送速率 × 利用率`
- 含义：链路实际平均数据传输能力
- 什么时候用：题目要求平均数据传输率时
- 常见误用：直接把带宽当平均传输速率

### HDLC 零比特填充

- 名称：零比特填充
- 表达式/方法：数据字段中每出现连续 5 个 1，就插入 1 个 0
- 含义：避免数据字段与标志字段混淆
- 什么时候用：HDLC 组帧/拆帧题
- 常见误用：把标志字段本身也拿去做比特填充

### 介质访问控制方法分类

- 名称：介质访问控制分类
- 表达式/方法：信道划分 = `TDMA/FDMA/CDMA`；随机访问 = `CSMA` 等
- 含义：划分法原则上避免冲突，随机访问法可能冲突
- 什么时候用：介质访问控制选择题
- 常见误用：把 CSMA 误归到信道划分方法

### CDMA 解码判定

- 名称：CDMA 解码
- 表达式/方法：`结果 = 接收序列 · 本站码片序列 / 码片长度`
- 含义：通过内积判断某站点发送的是 1 还是 0
- 什么时候用：CDMA 码片序列解码题
- 常见误用：忘记按每组码片长度分段计算

### 滑动窗口利用率

- 名称：滑动窗口利用率
- 表达式/方法：`U = min(1, W * Td / (Td + 2Tp + Tack))`
- 含义：窗口协议下发送方真正持续发送数据的比例
- 什么时候用：窗口大小、链路利用率、序号位数题
- 常见误用：忽略 ACK 时间或把窗口当序号位数

### 停等协议利用率

- 名称：停等协议利用率
- 表达式/方法：`U = Td / (Td + 2Tp + Tack)`
- 含义：发送一帧后必须等待确认返回，因此只有发送阶段在有效利用信道
- 什么时候用：停等协议利用率和帧长反推题
- 常见误用：把单向传播时延错当 RTT

### 通用滑动窗口约束

- 名称：发送窗口与接收窗口总约束
- 表达式/方法：`W_T + W_R <= 2^n`
- 含义：避免序号循环时新旧帧混淆
- 什么时候用：题目没有明确限定 GBN 或 SR，只给出位数与窗口关系时
- 常见误用：直接套 GBN 或 SR 的特例公式

### GBN 接收窗口

- 名称：GBN 接收窗口
- 表达式/方法：`W_R = 1`
- 含义：接收方只接收按序到达的下一帧
- 什么时候用：GBN 协议题
- 常见误用：把 GBN 的接收窗口也当成可变

### GBN 发送窗口上界

- 名称：GBN 发送窗口上界
- 表达式/方法：`W_T <= 2^n - 1`
- 含义：在 n 位序号空间下，发送窗口最大不能达到整个序号空间
- 什么时候用：GBN 窗口大小、序号位数反推题
- 常见误用：把上界写成 `2^n`

### SR 窗口上界

- 名称：SR 窗口上界
- 表达式/方法：`W_T(max) = W_R(max) = 2^(n-1)`
- 含义：SR 中发送窗口和接收窗口最大都只能到序号空间的一半
- 什么时候用：选择重传协议的位数、窗口上界题
- 常见误用：把 SR 误用成 GBN 的 `2^n - 1`

### 停等协议最大利用率

- 名称：停等协议最大利用率
- 表达式/方法：`U = Td / (Td + 2Tp + Tack)`
- 含义：一个停等周期里，有效发送数据时间所占的比例
- 什么时候用：停等协议利用率题，尤其是 ACK 不能忽略时
- 常见误用：漏掉 ACK 的发送时延

### 交换机转发依据

- 名称：交换机转发依据
- 表达式/方法：依据目的 MAC 地址查 MAC 地址表决定转发端口
- 含义：交换机在链路层根据目标物理地址转发帧
- 什么时候用：交换机、网桥、链路层转发题
- 常见误用：错把交换机当成根据 IP 地址转发

### CSMA/CD 最小帧长约束

- 名称：CSMA/CD 最小帧长约束
- 表达式/方法：`T帧 >= 2τ`
- 含义：发送方在发完前必须来得及检测到最远端冲突
- 什么时候用：最小帧长、最大距离、传播时延相关题
- 常见误用：忘记是往返传播时延而不是单程

### 距离与传播时延

- 名称：传播时延与距离关系
- 表达式/方法：`距离 = 传播速度 × 传播时延`
- 含义：信号跑多远取决于速度和传播时间
- 什么时候用：根据传播时间变化推距离变化题
- 常见误用：把往返时间直接当单程时间算距离

### CSMA/CA 确认机制

- 名称：CSMA/CA 确认机制
- 表达式/方法：接收方正确收到数据帧后返回 ACK
- 含义：通过确认帧判断发送是否成功
- 什么时候用：802.11、无线局域网、MAC 协议比较题
- 常见误用：把 ACK 机制错记到 CSMA/CD

### 集线器的域属性

- 名称：集线器的冲突域与广播域
- 表达式/方法：Hub 下所有站点同属一个冲突域，也同属一个广播域
- 含义：集线器不隔离冲突，也不隔离广播
- 什么时候用：Hub/交换机/路由器对比题
- 常见误用：把集线器误认为能划分冲突域

### 集线器带宽共享

- 名称：集线器带宽共享
- 表达式/方法：总带宽由所有主机共享
- 含义：多台主机连接同一集线器时，平均可用带宽会下降
- 什么时候用：共享式以太网带宽分配题
- 常见误用：误以为每台主机都独享端口速率

### 集线器转发方式

- 名称：集线器转发方式
- 表达式/方法：输入信号从除输入端口外的所有端口转发出去
- 含义：物理层扩散，不做地址学习
- 什么时候用：Hub 与交换机、网桥对比题
- 常见误用：误以为 Hub 会根据目的地址选端口

### 中继器/集线器共同点

- 名称：中继器与集线器共同点
- 表达式/方法：工作在物理层，可放大整形信号，但级联受限
- 含义：两者都不具备链路层转发能力
- 什么时候用：中继器/集线器判断题
- 常见误用：误以为可无限扩展网段

### p-坚持 CSMA

- 名称：p-坚持 CSMA
- 表达式/方法：监听到空闲后，以概率 p 发送，以概率 1-p 推迟一个时隙
- 含义：即使信道空闲，也可能暂不发送
- 什么时候用：CSMA 变体辨析题
- 常见误用：把它和 1-坚持 CSMA 混淆

### 二进制指数退避

- 名称：二进制指数退避
- 表达式/方法：第 i 次冲突后，随机从 `0 ~ (2^k - 1)` 中选取，`k = min(i, 10)`
- 含义：冲突越多，退避随机范围越大
- 什么时候用：以太网冲突后退避范围题
- 常见误用：忘记最大只增长到 10

### 网卡实现层次

- 名称：网卡实现层次
- 表达式/方法：网卡主要实现物理层和数据链路层功能
- 含义：既负责比特传输，也负责 MAC 子层功能
- 什么时候用：网卡/设备分层题
- 常见误用：误以为网卡只工作在物理层

### MAC 地址

- 名称：MAC 地址
- 表达式/方法：以太网地址 = MAC 地址 = 局域网硬件地址
- 含义：链路层地址，通常存于网卡
- 什么时候用：地址类型辨析题
- 常见误用：误以为通过域名解析获得

### 常见设备对冲突域的影响

- 名称：常见设备对冲突域的影响
- 表达式/方法：集线器不能分割冲突域；网桥、交换机、路由器都能分割冲突域
- 含义：判断设备能否隔离冲突
- 什么时候用：设备分层与冲突域题
- 常见误用：把“不能分割广播域”误记成“不能分割冲突域”

### IEEE 802 对应层次
- 名称：IEEE 802 标准分层
- 表达式/方法：IEEE 802 主要对应 OSI 的物理层和数据链路层
- 含义：局域网标准同时规定了介质/编码与 MAC/LLC
- 什么时候用：IEEE 802、LAN 标准层次判断题
- 常见误用：误以为只对应物理层

### VLAN 的广播域作用
- 名称：VLAN 的域划分能力
- 表达式/方法：VLAN 可把一个交换网络划分成多个逻辑广播域
- 含义：VLAN 不只隔离冲突域，也隔离广播域
- 什么时候用：VLAN 概念、交换机广播域题
- 常见误用：把 VLAN 错记成“不能隔离广播域”

### 以太网 MAC 服务类型
- 名称：MAC 服务类型
- 表达式/方法：无连接 + 不可靠
- 含义：不建立连接，不保证端到端可靠交付
- 什么时候用：MAC/TCP 服务对比题
- 常见误用：把运输层可靠性套到以太网 MAC

### 100BaseT 命名
- 名称：100BaseT
- 表达式/方法：100 = 100Mbps；Base = 基带；T = 双绞线
- 含义：快速以太网双绞线介质命名规则
- 什么时候用：以太网介质与命名题
- 常见误用：把 T 记成同轴电缆

### 交换机与 Hub 的域属性
- 名称：Hub / Switch 域划分
- 表达式/方法：Hub = 1 个冲突域 + 1 个广播域；Switch = 每端口 1 个冲突域 + 默认 1 个广播域
- 含义：Hub 共享，Switch 交换
- 什么时候用：冲突域/广播域数量题
- 常见误用：把交换机误认为不能分割冲突域

### 设备与冲突域/广播域
- 名称：常见设备域划分总表
- 表达式/方法：Hub 不能分冲突/广播；Bridge/Switch 能分冲突但默认不分广播；Router 两者都能分
- 含义：按层次判断设备隔离能力
- 什么时候用：设备分层与域划分综合题
- 常见误用：把“不能分广播域”误记成“不能分冲突域”
