# 408 计算机网络

## Goal

- 以 408 统考要求为边界，高效拿下计算机网络模块。
- 不追求泛泛了解网络，而是针对 408 的高频考点、易错计算题和综合题做定向训练。
- 最终达到：选择题稳、综合题会拆、公式和协议细节不混淆。

## Scope

- 按 408 常见范围学习：网络体系结构、物理层、数据链路层、网络层、运输层、应用层。
- 覆盖高频计算：时延/带宽相关、CRC、CSMA/CD、IP 编址与子网划分、路由聚合、分片、TCP 窗口与拥塞控制。
- 覆盖高频记忆：协议功能、报文字段、适用场景、层间关系。

## Non-Scope

- 不以工程部署为主线，不展开交换机/路由器厂商配置。
- 不把 Web 开发细节、Linux 网络命令作为当前重点。
- 不追求超纲前沿网络技术，先保证 408 得分效率。

## Outcome

- 能独立画出五层/四层体系结构并说明每层职责。
- 能熟练处理网络层、运输层相关计算题。
- 能把综合题拆成“概念判断 + 计算 + 协议解释”三部分作答。
- 能形成一套 408 计算机网络高频题型清单。

## Exam Targeting

- 依据近年 408 公开分析，计算机网络约 25 分。
- 常见结构是 8 道选择题加 1 道综合题，综合题多落在数据链路层、网络层、运输层，且常有交叉。
- 因此复习策略应是：应用层和体系结构保选择题正确率，数据链路层/网络层/运输层保综合题突破。
