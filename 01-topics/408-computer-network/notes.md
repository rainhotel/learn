## 第一课：计算机网络概述与性能指标

### 1. 三个容易混的概念

- 计算机网络：把分散的计算机连接起来，实现资源共享和信息传输的系统。
- 互联网：多个网络通过路由器互连形成的更大网络。
- 互连网：强调“网络的网络”，本质上就是很多网络连接而成。

记忆方式：

- 单个局部连接系统看“计算机网络”
- 多个网络互连形成整体看“互联网/互连网”

### 2. 为什么要分层

教材里的核心思想不是“好看”，而是“复杂系统分解”。

分层的价值：

- 各层功能相对独立
- 下层对上层提供服务
- 协议标准化后，不同厂商设备也能互通
- 某一层变化时，不必整体推倒重来

### 3. 协议、服务、接口的关系

这是 408 选择题高频混淆点。

- 协议：对等实体之间通信的规则，属于水平关系。
- 服务：下层通过层间接口向上层提供的功能，属于垂直关系。
- 接口：相邻两层交换信息的边界。

一句话记忆：

- 协议是“同层怎么说话”
- 服务是“下层能为上层做什么”
- 接口是“层和层从哪里交接”

### 4. 两种体系结构怎么记

#### OSI 七层

- 应用层
- 表示层
- 会话层
- 运输层
- 网络层
- 数据链路层
- 物理层

#### TCP/IP 常见教学五层

- 应用层
- 运输层
- 网络层
- 数据链路层
- 物理层

408 复习时一般按五层理解最顺手，但做题时要知道 OSI 七层的名字和对应关系。

### 5. 性能指标是第一轮必拿分点

#### 总时延

总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延

- 发送时延：把分组所有比特推上链路所需时间
- 传播时延：信号在信道中跑到对端所需时间
- 处理时延：节点处理分组所需时间
- 排队时延：分组在队列里等待所需时间

#### 发送时延

发送时延 = 分组长度 / 发送速率

关键点：

- 和数据长度有关
- 和链路距离无关

#### 传播时延

传播时延 = 信道长度 / 信号传播速率

关键点：

- 和距离有关
- 和数据长度无关

### 6. 带宽与吞吐量

- 带宽：链路理论上能承载的最高数据率
- 吞吐量：单位时间内实际成功传输的数据量

不要混淆：

- 带宽是上限能力
- 吞吐量是实际表现

### 7. 这一课的考试意义

这一章虽然不是综合题主战场，但它决定你后面很多章节是否会混：

- 不懂分层，应用层到运输层的协议关系会乱
- 不懂时延，后面链路层和网络层相关题会反复错
- 不懂协议和服务，选择题会丢分很冤

## Key Memory Lines

### 体系结构

- 协议是水平的，服务是垂直的。
- 分层的核心价值是解耦和标准化。
- 发送时延看数据长度，传播时延看传播距离。

### 数据链路层

- CRC 负责检错，不负责纠错。
- CSMA/CD 的核心是边发边听。
- 最小帧长的本质是“来得及发现冲突”。

### 网络层

- IP 只提供尽最大努力交付。
- CIDR 的核心是无类编址和最长前缀匹配。
- 分片偏移量按 8 字节计。

### 运输层

- UDP 简单、无连接、不保证可靠。
- TCP 面向连接、可靠、字节流。
- 流量控制管接收方，拥塞控制管网络。

### 应用层

- DNS 解决名字到 IP 的映射。
- HTTP 本身无状态。
- DHCP 解决主机动态获取网络配置。

## 第三课：滑动窗口、介质访问控制与 HDLC

### 一、这一组题在 408 里的考点位置

这 6 题都属于数据链路层高频选择题范围，集中考：

- 滑动窗口协议的确认与重传机制
- GBN 与 SR 的区别
- 信道利用率与窗口大小
- 介质访问控制方法
- HDLC 的零比特填充

### 二、先把底层原理记住

#### 1. GBN 的本质

- 发送方可以连续发送多个帧
- 接收方只按序接收
- 确认是累计确认
- 一旦超时，从最早未确认帧开始，把后面的已发未确认帧全部重传

识别关键词：

- 累计确认
- 回退重传

#### 2. SR 的本质

- 每一帧单独确认
- 每一帧单独计时
- 哪一帧超时就只重传哪一帧
- 接收方可以缓存失序帧

识别关键词：

- 单独确认
- 选择重传

#### 3. 信道利用率题怎么想

窗口协议的目标，是让发送方在第一帧确认回来前尽量一直有帧可发。

所以要先算：

- 一个数据帧发送时延 `Td`
- 一个确认返回周期需要多久
- 窗口内帧能不能覆盖这个周期

#### 4. 介质访问控制方法怎么分

教材里常分两大思路：

- 信道划分：TDMA、FDMA、CDMA
- 随机访问：ALOHA、CSMA、CSMA/CD、CSMA/CA

随机访问的本质是“大家都可能抢着发”，所以可能冲突。

#### 5. HDLC 的零比特填充

规则只有一句话：

- 在数据部分中，只要出现连续 5 个 1，就自动插入 1 个 0

目的：

- 避免把数据误判成标志字段 `01111110`

### 三、这 6 题的统一做题模板

#### GBN 重传题

- 看最大的已确认帧到哪
- 从最早未确认帧开始往后全重传

#### SR 重传题

- 看哪些帧超时
- 超时几帧就重传几帧

#### 利用率/窗口题

- 先算 `Td`
- 再算确认返回周期
- 再判断窗口至少能装多少帧

#### HDLC 填充题

- 扫描数据字段
- 每遇到连续 5 个 1，插入 0

## 第四课：CDMA、滑动窗口利用率与停等协议

### 一、这一组题在 408 里的位置

这 3 题分别对应：

- CDMA 的码片序列与解码
- 滑动窗口协议的链路利用率与序号位数
- 停等协议的信道利用率

它们都属于“原理理解 + 计算”型选择题。

### 二、CDMA 的底层原理

CDMA 不是把时间分开，也不是把频率分开，而是给每个站点分配一组不同的码片序列。

核心思想：

- 发送 1：发送自己的码片序列
- 发送 0：发送自己码片序列的反码
- 多个站点同时发送时，链路上的信号是它们的“叠加”
- 接收方想知道某个站点发了什么，就用收到的序列和该站点的码片序列做内积

如果内积结果：

- 为正：该站点发 1
- 为负：该站点发 0
- 为 0：说明该站点当前没发或正负抵消

### 三、滑动窗口利用率的底层原理

无论是 GBN 还是一般滑动窗口，核心目标都是：

- 让发送方在等待确认的这段时间里，仍然能持续发帧

所以判断链路利用率，关键不是先背公式，而是先比较两个时间：

1. 窗口内所有帧发完需要多久
2. 第一帧确认返回需要多久

如果窗口足够大，发送方就不会空等，利用率就高。

### 四、停等协议的底层原理

停等协议最简单：

- 发 1 帧
- 等确认
- 再发下一帧

因此它利用率低的根本原因是：

- 发送时间往往很短
- 等待确认返回时间往往很长

所以停等协议利用率的本质是：

- 有多少时间真正用来发数据
- 有多少时间在空等

## 第五课：滑动窗口的三个核心约束

### 一、这一组题在 408 里的位置

这组题几乎全在考“窗口协议的边界条件”，属于 408 数据链路层非常典型的套路题。

核心考点只有三个：

- 发送窗口和接收窗口的关系
- GBN 的累计确认
- 帧序号位数与窗口上界的关系

### 二、先讲原理

#### 1. 为什么窗口大小不能乱设

窗口协议里，帧序号是循环使用的。

比如 3 位编号时，序号空间只有：

- 0,1,2,3,4,5,6,7

用完又会回到 0。

这就带来一个风险：

- 接收方看到一个 0
- 它怎么知道这是“老的 0”还是“新的一轮 0”？

所以窗口大小必须受限制，目的就是：

- 不让新旧帧混淆

#### 2. 一般滑动窗口协议的总约束

如果序号空间大小为 `2^n`，则要避免新旧帧混淆，需要满足：

- `发送窗口 + 接收窗口 <= 2^n`

这是“通用滑动窗口”题最核心的一条。

#### 3. 为什么 GBN 和 SR 的约束不同

##### GBN

- 接收窗口恒为 1
- 因为接收方只按序接收，失序帧直接丢弃
- 所以 GBN 重点看发送窗口上界：
  - `W_T <= 2^n - 1`

##### SR

- 发送方和接收方都可能保留多个未完成帧
- 为避免新旧帧混淆，通常要求：
  - `W_T <= 2^(n-1)`
  - `W_R <= 2^(n-1)`

#### 4. GBN 的确认机制为什么会让题目变简单

GBN 用的是累计确认。

也就是说：

- 如果已经收到对 5 号帧的确认
- 那么 0 到 5 都默认已经确认
- 最早未确认帧就是 6

所以 GBN 重传题本质上就是：

- 找“最大的已确认帧”
- 然后从下一个开始重传

#### 5. 位数和窗口的反推逻辑

如果题目问：

- 给定窗口大小，至少需要多少位序号

你就反过来找最小的 `n`，使得窗口约束成立。

如果题目问：

- 给定位数，最大窗口是多少

你就直接用上界公式。

## 第六课：SR 窗口上界与停等协议信道利用率

### 一、这一组题在 408 里的位置

这两题都是数据链路层的经典选择题：

- SR 的窗口上界
- 停等协议的最大信道利用率

它们共同在考：

- 序号空间为什么限制窗口大小
- 等待确认为什么会降低链路利用率

### 二、SR 的底层原理

SR（选择重传）和 GBN 最大的不同，不是在“能不能发很多帧”，而是在：

- GBN：接收方只接收按序帧，失序帧丢掉
- SR：接收方可以缓存失序帧

正因为 SR 会缓存失序帧，所以如果窗口开得过大，当序号循环回来时，接收方就可能分不清：

- 这是旧帧
- 还是新一轮的同序号帧

为了避免这种混淆，SR 的发送窗口和接收窗口都不能超过序号空间的一半。

所以 SR 的核心上界是：

- `W_T <= 2^(n-1)`
- `W_R <= 2^(n-1)`

### 三、停等协议利用率的底层原理

停等协议最简单：

- 发 1 帧
- 等 ACK
- 再发下一帧

所以一个完整周期包括：

1. 发送数据帧
2. 数据传播到对端
3. 发送确认帧
4. 确认传播回来

链路真正用于传“有效数据帧”的时间，只是其中的一部分。

因此停等协议的利用率本质是：

- 发数据所占时间 / 一个完整发送确认周期时间

如果 ACK 也不短，就不能忽略 ACK 的发送时延。

## 第七课：交换机转发、CSMA/CD 与 CSMA/CA

### 一、这一组题在 408 里的位置

这 3 题分别对应：

- 交换机在链路层依据什么地址转发
- CSMA/CD 为什么要求最小帧长
- CSMA/CA 为什么要确认帧

它们都属于链路层和 MAC 子层的高频概念题。

### 二、交换机转发的底层原理

以太网交换机工作在数据链路层。

它处理的协议数据单元是：

- 帧

帧里最关键的地址是：

- 源 MAC 地址
- 目的 MAC 地址

交换机做转发决策时，真正看的不是 IP 地址，而是：

- 目的 MAC 地址

原因很简单：

- 交换机只负责把帧转到正确端口
- 它维护的是 MAC 地址表
- 不做网络层的路由判断

### 三、CSMA/CD 最小帧长的底层原理

CSMA/CD 的关键思想是：

- 边发边听
- 如果发生冲突，发送方必须在自己“还没发完”时就能检测到冲突

所以最小帧长的本质不是一个死数字，而是一个约束：

- 一帧的发送时间 >= 最远两站之间的往返传播时延

也就是：

- `T帧 >= 2τ`

如果最小帧长缩短了，那么为了仍然满足冲突检测，链路最大长度也必须缩短。

### 四、CSMA/CA 为什么需要确认

在无线网络中，站点很难像有线网那样边发边检测冲突，所以不能像 CSMA/CD 那样靠“检测碰撞”来补救。

因此 CSMA/CA 的核心思路变成：

- 尽量避免冲突
- 发完后由接收方返回 ACK
- 发送方通过是否收到 ACK 来判断是否成功

所以：

- CSMA/CD 靠检测冲突
- CSMA/CA 靠确认接收

## 第九课：集线器、中继器、冲突域与广播域

### 一、这一组题在 408 里的位置

这组题主要考物理层设备与共享式以太网的基本性质：

- 集线器连接的主机处于什么域
- 集线器带宽如何共享
- 集线器转发数据的方式
- 中继器与集线器的共同点和限制

### 二、先讲底层原理

#### 1. 集线器本质上是什么

集线器（Hub）本质上就是一个多端口中继器。

它工作在：

- 物理层

它不理解帧，也不识别 MAC 地址，更不会做转发表。

它干的事情非常“原始”：

- 某个端口收到电信号
- 放大、整形
- 发到其他端口

所以它没有“智能转发”，只有“信号复制扩散”。

#### 2. 为什么集线器下所有主机同属一个冲突域

因为它只是共享一根逻辑总线。

- 所有人都在争用同一条传输信道
- 任意两台同时发，都可能冲突

所以整个 Hub 下挂的所有主机：

- 同属一个冲突域

#### 3. 为什么集线器下所有主机也同属一个广播域

广播帧进入集线器后，会被扩散到所有其他端口。

因此：

- 所有主机都能收到同一个广播
- 所以它们也同属一个广播域

#### 4. 为什么集线器的总带宽要共享

集线器不为每个端口提供独立交换带宽。

它只是让多个主机共享同一段介质，因此：

- 总带宽固定
- 多台主机一起用时，只能分摊这份带宽

#### 5. 中继器和集线器为什么都属于物理层

因为二者都不处理帧，不看 MAC，也不看 IP。

它们只处理：

- 比特信号
- 电信号/光信号的放大与整形

所以它们都属于物理层设备。

#### 6. 为什么不能无限级联中继器或集线器

虽然它们能放大整形信号，但不能无限制串联。

原因是：

- 时延会累积
- 冲突检测会受到影响
- 网络标准对冲突域跨度有限制

所以“网段数量不受限制”这种说法通常是错误的。

## 第十课：CSMA 变体、二进制退避、网卡与 MAC 地址

### 一、这一组题在 408 里的位置

这 4 题主要考：

- CSMA 三种坚持策略
- 以太网二进制指数退避算法
- 网卡工作的层次
- MAC 地址的基本概念

### 二、先讲底层原理

#### 1. 三种 CSMA 的核心区别

CSMA 共同点：

- 先监听信道
- 再决定发不发

不同点在“监听到空闲后怎么办”：

- 1-坚持 CSMA：一旦空闲，立即发送
- 非坚持 CSMA：若忙则退避；重新监听到空闲时再立即发送
- p-坚持 CSMA：若空闲，也不是一定立刻发，而是以概率 p 发送，以概率 1-p 再等一个时隙

所以只有 p-坚持 CSMA 会出现：

- 明明侦听到空闲，但仍然可能暂时不发

#### 2. 二进制指数退避为什么要“越撞越等”

以太网冲突后，不会立刻所有站点同时重发，否则会再次冲突。

所以每个站点会：

- 等一个随机时间
- 再尝试发送

冲突次数越多，随机范围越大，这样更容易把站点错开。

规则是：

- 第 i 次冲突后，随机从 `0 ~ (2^k - 1)` 里选一个
- 其中 `k = min(i, 10)`

#### 3. 网卡实现了哪些层的功能

网卡不是纯物理层设备。

它至少实现：

- 物理层：比特发送、接收、电气/光学接口
- 数据链路层：成帧、MAC 地址识别、差错检测等

所以网卡通常对应：

- 物理层 + 数据链路层

#### 4. MAC 地址到底是什么

MAC 地址就是以太网地址，也叫局域网硬件地址。

特点：

- 属于链路层地址
- 一般烧录或存放在网卡中
- 不靠 DNS 解析得到

DNS 解析得到的是：

- 域名对应的 IP 地址

不是 MAC 地址。

## 更正记录：第25题冲突域判断

### 题目结论

- “下列不能分割冲突域的设备是()” 的正确答案是：`集线器`

### 为什么容易错

- 这类题最容易把“冲突域”和“广播域”混在一起。
- 交换机和网桥默认不能分割广播域，但它们可以分割冲突域。
- 集线器既不能分割冲突域，也不能分割广播域。

### 一次记住

- 集线器：不能分割冲突域
- 网桥：能分割冲突域
- 交换机：能分割冲突域
- 路由器：能分割冲突域

## 链路层设备与以太网补充归档（2026-03-29）

### IEEE 802 与 OSI
- IEEE 802 系列标准主要覆盖局域网/城域网的物理层与数据链路层。
- 数据链路层里重点是 LLC 与 MAC。

### VLAN 的本质
- VLAN 是基于交换机实现的逻辑划分，不是物理拆网。
- VLAN 的核心作用之一就是划分广播域。

### 以太网 MAC 服务
- 以太网 MAC 提供的是无连接、不可靠服务。
- 它不负责像 TCP 那样的端到端确认重传。

### 100BaseT 命名
- `100` 表示 100 Mbps。
- `Base` 表示基带传输。
- `T` 表示双绞线（Twisted Pair）。

### 设备层次与域划分速记
- 集线器：物理层；不能分割冲突域；不能分割广播域。
- 网桥：数据链路层；能分割冲突域；默认不能分割广播域。
- 交换机：数据链路层；每端口一个冲突域；默认不能分割广播域；配合 VLAN 可隔离广播域。
- 路由器：网络层；既能分割冲突域，也能分割广播域。

### Hub 与 Switch 做题入口
- Hub：信号扩散、共享带宽、一个冲突域、一个广播域。
- Switch：按 MAC 学习转发、每端口一个冲突域、默认整个交换网络一个广播域。

### 拓扑题注意点
- 物理层收到信号，不等于链路层真正接收该帧。
- 连在 Hub 上的其他主机，常常会在物理层“听到”信号，但链路层会因为目的 MAC 不匹配而丢弃。
