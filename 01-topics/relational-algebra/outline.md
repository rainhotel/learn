# Relational Algebra Mastery Outline

## Phase 0: Positioning And Scope

- 关系代数是关系数据库的“操作语言”。
- 它关注的是“从哪些行、哪些列、按什么规则生成新关系”。
- 先修知识：关系、元组、属性、候选码的基础概念。

## Phase 1: Map The Field

- 识别常见符号：`σ`、`Π`、`×`、`⋈`、`÷`
- 分清行运算、列运算、组合运算、全称匹配运算
- 看懂结果表的属性顺序和去重规则

## Phase 2: Foundations

- 学懂关系和二维表之间的对应关系
- 学懂“选择是筛行，投影是取列”
- 学懂投影会去重，而选择不会改列
- 学懂除法表达的是“对 D 中每一个都满足”

## Phase 3: Core Mechanisms

- 能从左到右拆公式
- 能在笛卡儿积后用属性位置或属性名做条件筛选
- 能把自然连接理解成“先配对再合并公共列”
- 能把除法理解成“找完整覆盖集合的前缀属性”

## Phase 4: Patterns And Trade-Offs

- 看到 `σ(condition)(R)` 就先想“留哪些行”
- 看到 `Π(attrs)(R)` 就先想“保留哪些列，再去重”
- 看到 `R × S` 后面接选择，就警惕这通常是在手工构造连接
- 看到 `R ÷ D` 就先找被保留的属性，再检查是否覆盖 D 的全部元组

## Phase 5: Deliberate Practice

- 练熟单步运算题
- 练熟“笛卡儿积 + 选择 + 投影”的复合题
- 练熟自然连接和除法

## Phase 6: Build Something Real

- 把一组 SQL 查询翻译成关系代数
- 把同一题目分别用连接和除法表述

## Phase 7: Explain, Teach, Defend

- 能用口头语言解释每个运算符
- 能讲清楚“为什么投影要去重”
- 能讲清楚“为什么除法表示全称条件”

## Phase 8: Frontier And Taste

- 了解关系代数与 SQL 的表达差异
- 理解逻辑查询、代数优化和执行计划之间的联系

## Phase Exit Criteria

### 每一阶段结束前都检查

- 我能不能用一句话解释每个运算符？
- 我能不能不看答案写出中间表？
- 我能不能指出这道题在考“存在”还是“全部”？
- 我能不能把符号翻译成自然语言？
